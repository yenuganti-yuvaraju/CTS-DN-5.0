# Module 10 - Continuous Integration and Continuous Delivery (CI/CD)

## Objective

The objective of this module is to understand the fundamentals of Continuous Integration (CI), Continuous Delivery (CD), Continuous Deployment, and the role of DevOps in modern software development. This module also introduces popular CI/CD tools used in the software industry.

---

# Introduction to DevOps

DevOps is a software development approach that combines Development (Dev) and Operations (Ops). It improves collaboration between development and operations teams to build, test, deploy, and maintain applications more efficiently.

## Goals of DevOps

- Improve collaboration between developers and operations teams.
- Automate software development and deployment.
- Deliver software faster and more reliably.
- Reduce deployment failures.
- Improve software quality.
- Enable continuous monitoring and feedback.

---

# Benefits of DevOps

- Faster software delivery.
- Better collaboration among teams.
- Reduced manual effort through automation.
- Higher application reliability.
- Faster bug fixing.
- Continuous improvement of software.

---

# Continuous Integration (CI)

Continuous Integration is a software development practice where developers frequently merge their code into a shared repository.

Whenever code is pushed:

- Source code is compiled.
- Automated tests are executed.
- Code quality is checked.
- Build artifacts are generated.

This helps identify bugs early in the development process.

### Advantages of Continuous Integration

- Early bug detection.
- Reduced integration problems.
- Automated testing.
- Faster development.
- Improved code quality.

---

# Continuous Delivery (CD)

Continuous Delivery is the practice of automatically preparing applications for deployment after successful testing.

In Continuous Delivery:

- Code is built automatically.
- Tests are executed.
- Deployment package is created.
- Deployment to production requires manual approval.

### Benefits

- Reliable software releases.
- Faster release cycles.
- Reduced deployment risks.
- Better customer satisfaction.

---

# Continuous Deployment

Continuous Deployment is an extension of Continuous Delivery.

After all automated tests pass successfully, the application is automatically deployed to production without manual approval.

### Advantages

- Faster software delivery.
- Immediate deployment.
- Reduced manual intervention.
- Continuous updates for users.

---

# Difference between CI, Continuous Delivery and Continuous Deployment

| Feature | CI | Continuous Delivery | Continuous Deployment |
|----------|----|--------------------|------------------------|
| Code Integration | Yes | Yes | Yes |
| Automated Build | Yes | Yes | Yes |
| Automated Testing | Yes | Yes | Yes |
| Manual Approval | No | Yes | No |
| Automatic Production Deployment | No | No | Yes |

---

# CI/CD Pipeline

A CI/CD pipeline automates the complete software delivery lifecycle.

Typical pipeline stages:

1. Write Code
2. Commit Code to Git Repository
3. Build Application
4. Execute Automated Tests
5. Perform Code Quality Checks
6. Package Application
7. Deploy to Staging Environment
8. Manual Approval (Continuous Delivery)
9. Deploy to Production
10. Monitor Application

# Popular CI/CD Tools

## Jenkins

- Open-source automation server.
- Supports build automation.
- Large plugin ecosystem.
- Widely used in enterprise applications.

---

## GitHub Actions

- Built into GitHub.
- Automates build, testing and deployment.
- Uses YAML workflow files.
- Easy integration with GitHub repositories.

---

## GitLab CI/CD

- Integrated within GitLab.
- Automates testing and deployment.
- Supports Docker and Kubernetes.

---

## CircleCI

- Cloud-based CI/CD platform.
- Fast build execution.
- Easy integration with GitHub and Bitbucket.
